# AI-Пульс — план опросника

## Концепция

Опросник для команды о том, как они используют AI-инструменты. Цель — собрать реальную картину (инструменты, кейсы, боли, идеи), не провести аудит. Должен ощущаться как пряник, а не кнут.

**Стратегическая цель:** измерять динамику AI-нативности команды. Планируется повторять с теми же core-вопросами через 1–2 месяца для отслеживания изменений.

Сам инструмент должен быть AI-нативным по форме — пример того, о чём спрашивает.

---

## UX и форма

- Видимая структура разделов с самого начала — человек видит объём, тревога снижается
- Прогресс-бар в виде динозаврика в стиле Chrome Dino — бежит и прыгает по мере прохождения
- Каждый вопрос поддерживает несколько форматов ответа: текст, голос (Web Speech API с расшифровкой), загрузка файла/скриншота
- Drag & drop для скриншотов
- Фирменный стиль компании — цвета, шрифты, логотип, снятые с сайта (ссылка будет передана отдельно)
- Финальный экран: AI-summary — карточка с архетипом и коротким портретом
- Мета-вопрос и "кто заполнял" — отдельный финальный экран с другой атмосферой

---

## Вводный экран

Один связный абзац (без буллетов). Тон — коллега к коллеге. Признаём, что анкета — это время, объясняем зачем: подписки, помощь с задачами, накопление практик. "Пишите сырее, а не красивее."

В конце абзаца — курсивная сноска: *Ну, или попросите своего агента заполнить эту форму. Только «сырее», а не красивее.*

Имя — обязательное, спрашивается в конце.

## Структура вопросов (6 разделов + финал)

### Раздел 1 — Инструменты

Описание раздела включает: это и про дженерик-модели (Claude, GPT, Gemini…) и про специализированные инструменты (Cursor, Lovable, Sora…).

**Список инструментов (плоский, 20 позиций):**
ChatGPT · Claude · Claude Code · Gemini · DeepSeek · Grok · Microsoft Copilot · GitHub Copilot · Cursor · Codeium / Windsurf · Perplexity · NotebookLM · ElevenLabs · Whisper · Midjourney · Bolt · Replit · Lovable · Sora · Kling

**Матрица на каждый инструмент:**

Первый выбор (radio):
- Использую
- Пробовал(а), не прижилось → открывается поле "Что пошло не так?"
- Не использую → больше ничего не показываем
- Хочу попробовать → больше ничего не показываем

Если "Использую" — открываются три под-вопроса:
- **Способ оплаты:** Корп. подписка / Бесплатная версия / Плачу сам / Хочу покрытие от компании
- **Частота:** Много раз в день / Несколько раз в неделю / Раз в неделю и реже / Только пробовал(а)
- **Как подключаешься (чекбоксы, несколько):** Веб / Приложение / Десктоп-клиент / Терминал (CLI) / API / Интеграция в IDE

**Usage snapshot в конце раздела:**
Если основной — Claude: `/insights` → приложить файл. Если другой: скриншот дашборда из настроек.

**Самообучающийся список:** через "другое" → нормализация Claude API → появляется у следующих при 2+ упоминаниях.

Личное использование: убрано.

---

### Раздел 2 — Задачи (CORE — не менять между итерациями)

*Вопрос про частоту убран как отдельный — частота теперь собирается per-tool в матрице Раздела 1.*

**Для каких задач используешь AI?**
Мультивыбор, сгруппированный по workflow-фазам — включая, но не ограничиваясь вопросами Феликса:

- 📋 **Планирование:** PRDs, estimation, breakdown задач, исследование, мозгоштурм
- ⚙️ **Реализация:** code gen, refactoring, debugging, ревью кода
- 🧪 **Тестирование:** генерация тестов, QA, тест-кейсы
- 🗂️ **За пределами кода:** документация, переводы, коммуникации, emails, найм, ops, дизайн, данные, обучение
- ✏️ **Что-то ещё:** открытое поле (текст)

---

### Раздел 3 — Глубина практики (CORE)

- **Топ-скиллы:** "Есть ли у тебя промпты или инструкции, которые используешь регулярно?" (да / нет) → если да: "Приложи или опиши один-два из своего топа" (текст / голос / файл)
- Используешь ли API / автоматизации / интеграции? (чекбокс)

*Убрано: "AI в редакторе" (это собирается в матрице Раздела 1 через "Интеграция в IDE") и шкала встроенности (биас, заменяется per-tool частотой из Раздела 1).*

---

### Раздел 4 — Кейс (CORE)

- Расскажи про момент, когда AI реально помог тебе в работе (текст / голос / скриншот / файл)
- Хочешь поделиться этим кейсом с командой? (да / анонимно / нет)
  - При "да" или "анонимно" — опциональный блок ниже:

    > *А был ли классный момент вне работы? Необязательно что-то масштабное — мелкие приколдесы тоже считаются.*
    > (текст / голос — опционально, раскрывается по клику)

---

### Раздел 5 — Ограничения и желания (CORE)

**Боли:**
- Где AI тебя подводил или разочаровал? (текст / голос)
- Насколько часто упираешься в ограничения AI? (шкала 1–5)

**Блокеры адопшена в команде:**
Что мешает более широкому использованию AI у вас? (мультивыбор)

Технические:
- [ ] Нет доступа к нужному инструменту
- [ ] Слишком дорого (нет бюджета)
- [ ] Ограничения безопасности / политики компании
- [ ] Технические ограничения среды (сети, VPN, устройства)
- [ ] Нет интеграции с нашим стеком

Человеческие:
- [ ] Не хватает навыков / не знаю, с чего начать
- [ ] Нет времени разбираться
- [ ] Нужен живой гайданс — кто-то из команды, кто покажет
- [ ] Нет понимания, что именно можно делегировать AI
- [ ] Скептицизм или сопротивление в команде
- [ ] Другое (поле)

**Гайданс:**
- "Есть задачи, где ты чувствуешь, что AI мог бы помочь — но не уверен как? Или хотел бы, чтобы кто-то показал?" (да / нет)
  - Если да: открытое поле — "Я хотел бы разобраться в / получить МК по…" (текст / голос)

---

### Финал — Summary + архетип

Генерация через Claude API: короткая карточка с архетипом и описанием на основе ответов.

**Архетипы:**
- 🦕 **Исследователь** — только начинает, использует редко и поверхностно
- 🚀 **Практик** — регулярно, по делу, но в рамках одного-двух инструментов
- ⚡ **Интегратор** — широкий стек, системный подход, уже меняет процессы
- 🧠 **Амбассадор** — всё вышеперечисленное плюс готов делиться с командой

---

### Финальный экран — мета + авторство

Другая атмосфера, чуть темнее.

**Заголовок:** *"Последний вопрос — и он про саму анкету."*

**Текст:** *"Ты только что прошёл опросник об AI. Он сделан вот так. Если бы это была твоя задача — как бы ты её решил?"*
Только открытое поле (текст или голос).

**Кто заполнял (обязательное):**
- ○ Сам(а) — без AI
- ○ Сам(а), но спрашивал(а) AI как сформулировать
- ○ Спрашивал(а) у агента про кейсы и инструменты
- ○ Дал(а) задачу заполнить — проверил(а) и поправил(а)
- ○ Я агент — меня попросили пройти и отправить опрос

## Метрики и scoring

### Числовые метрики (для аналитики и сравнения итераций)

| Метрика | Как считается | Источник |
|---|---|---|
| **AI Frequency Score** | 0–4 по шкале | Раздел 2 |
| **Stack Breadth (work)** | количество используемых инструментов | Раздел 1 |
| **Stack Breadth (personal)** | количество используемых лично | Раздел 1 |
| **Use Case Diversity** | количество отмеченных типов задач | Раздел 2 |
| **Depth Score (0–10)** | сохранённые скиллы + API/интеграции + AI в редакторе | Раздел 3 |
| **Workflow Integration** | 1–5 по шкале Лайкерта | Раздел 3 |
| **Sharing Propensity (0–3)** | отметка "поделиться" + файл + содержательность кейса | Раздел 4 |
| **Friction Score** | 1–5 | Раздел 5 |
| **Demand Signal (0–2)** | заполнено ли "что хочу уметь" + конкретность | Раздел 5 |
| **Personal Investment** | инструменты со статусом "плачу сам" / "хочу от компании" | Раздел 1 |
| **Dropped Tools Rate** | количество инструментов со статусом "не прижилось" | Раздел 1 |
| **Tool Discovery Rate** | новые инструменты через "другое" | Раздел 1, между итерациями |
| **Agent Fill Rate** | доля ответов "ассистент заполнил / частично" | Финальный экран |

### AI-Native Index

Среднее по всем баллам, нормализованное 0–100. Сравниваем между итерациями.

### Промпт-классификатор для открытых полей

**Зафиксировать сейчас — использовать без изменений во всех итерациях.** Версионируется отдельно.

Классифицируем:
- **Кейсы** → тип задачи / инструмент / уровень сложности (rote / augmentation / transformation)
- **Боли** → тип проблемы (качество модели / интерфейс / стоимость / безопасность / навык)
- **Скиллы** → домен (код / письмо / анализ / дизайн / др.)
- **Dropped tools** → причина отказа (качество / стоимость / неудобство / нашёл лучше / др.)
